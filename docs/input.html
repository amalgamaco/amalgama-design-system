Input · Campo de texto

Los campos de texto permiten al usuario ingresar y editar texto. Se usan en formularios, dialogos y pantallas donde se requiere captura de informacion.

Instalacion

Copia input.tsx a tu proyecto en components/ui/input.tsx

import * as React from "react"
import { cn } from "@/lib/utils"

export interface InputProps
  extends React.InputHTMLAttributes<HTMLInputElement> {
  label?:       string
  required?:    boolean
  hint?:        string
  error?:       string
  leadingIcon?: React.ReactNode
  trailingIcon?: React.ReactNode
  maxLength?:   number
}

const Input = React.forwardRef<HTMLInputElement, InputProps>(
  ({ className, label, required, hint, error, leadingIcon, trailingIcon,
     maxLength, value, defaultValue, onChange, type, ...props }, ref) => {

    const [charCount, setCharCount] = React.useState(
      String(value ?? defaultValue ?? "").length
    )

    const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
      setCharCount(e.target.value.length)
      onChange?.(e)
    }

    const input = (
      <div className="field-input-wrapper">
        {leadingIcon  && <span className="field-leading-icon">{leadingIcon}</span>}
        <input
          type={type}
          ref={ref}
          className={cn(
            "field-input",
            leadingIcon  && "has-leading",
            trailingIcon && "has-trailing",
            className
          )}
          value={value}
          defaultValue={defaultValue}
          maxLength={maxLength}
          onChange={handleChange}
          aria-invalid={!!error}
          aria-describedby={error ? "field-err" : hint ? "field-hint" : undefined}
          {...props}
        />
        {trailingIcon && <span className="field-trailing-icon">{trailingIcon}</span>}
      </div>
    )

    return (
      <div className={cn("field-group", error && "is-error")}>
        {label && (
          <label className="field-label">
            {label}
            {required && <span className="field-required" aria-hidden="true">*</span>}
          </label>
        )}
        {input}
        {(hint || error || maxLength) && (
          <div className="field-supporting">
            {error
              ? <span className="field-error-msg" id="field-err" role="alert">
                  <svg width="12" height="12" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2.5">
                    <circle cx="12" cy="12" r="10"/><line x1="12" y1="8" x2="12" y2="12"/><line x1="12" y1="16" x2="12.01" y2="16"/>
                  </svg>
                  {error}
                </span>
              : hint && <span className="field-hint" id="field-hint">{hint}</span>
            }
            {maxLength && (
              <span className={cn("field-char-count", charCount > maxLength && "is-over")}>
                {charCount}/{maxLength}
              </span>
            )}
          </div>
        )}
      </div>
    )
  }
)
Input.displayName = "Input"

export { Input }

Variante

Embassy emplea un unico tipo de campo de texto: el campo con contorno (Outlined). A diferencia de Material Design, que ofrece tambien la variante Filled, Embassy usa exclusivamente el contorno para maximizar la claridad visual en superficies claras y oscuras. 

<div class="field-group">
  <label class="field-label">Nombre completo</label>
  <input class="field-input" type="text" placeholder="Ej: Ana Garcia">
</div>

Estados

Cada estado comunica una condicion diferente al usuario. El contorno, la etiqueta y el texto de soporte cambian de color segun el estado activo.

Habilitado

Estado por defecto. El contorno usa --color-outline-variant y la etiqueta usa --color-on-surface.

Modo claro
Modo oscuro
<div class="field-group">
  <label class="field-label">Nombre</label>
  <input class="field-input" placeholder="Escribe aqui...">
</div>

Hover

Al posicionar el cursor sobre el campo, el contorno se refuerza con --color-outline (#9FA3AE), mas prominente que el estado habilitado. Solo aplica cuando el campo no esta en foco ni deshabilitado.

Modo claro — simulacion
Modo oscuro — simulacion

Focus

Cuando el campo recibe el foco del teclado o del clic, el contorno cambia a --interactive (#4F80FF) y aparece un anillo de 3px (--color-focus-ring). La etiqueta tambien cambia a --interactive. Haz clic en el campo para ver el efecto.

Modo claro — haz clic en el campo
Modo oscuro — haz clic en el campo

Error

Indica que el valor ingresado no es valido. El estado de error se activa agregando la clase .is-error al contenedor .field-group. El contorno, la etiqueta y el icono final cambian a --color-error. El mensaje de error reemplaza al texto de soporte normal. En estado de error con foco, el anillo usa un tono rojizo (rgba(253,58,69,.12)).

Deshabilitado

El campo deshabilitado no puede ser editado ni recibir foco. El fondo cambia a --color-surface-variant y el texto y contorno usan los tokens de estado deshabilitado. El cursor cambia a not-allowed.

Modo claro
Este campo no puede modificarse.
Modo oscuro
Este campo no puede modificarse.

  <div class="field-supporting">
    <span class="field-hint">Este campo no puede modificarse.</span>
