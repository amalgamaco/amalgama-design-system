Search

Búsqueda de contenido. Dos variantes: search bar (campo píldora, --radius-full, fondo surface-container-high + borde sutil — 40px en .search-row desktop, 56px standalone mobile/hero) y search view (vista expandida con resultados, en modo docked o full-screen).

CSS: css/components/search.css · React: components/ui/search.tsx (SearchBar, SearchView y sub-componentes)

Patrón canónico para búsqueda con acciones: .search-row + .search-icon-btn. Reusar en cualquier superficie que combine búsqueda y controles contextuales.

Cuándo usar

Cuándo usarsearch-bar en .search-row (desktop, 40px) con .search-icon-btn para acciones contextuales. search-bar standalone (56px, mobile o hero de página) cuando no hay acciones adyacentes.
Cuándo noUsar como campo de formulario — para eso está .form-input. Acompañar con botones de texto — usar .search-icon-btn en su lugar.
Reemplaza aInputs de búsqueda legacy, toolbar con search-field y botones de texto.

Search + botones de acción

Patrón canónico de Embassy para cualquier contexto toolbar-like. Usar .search-row como wrapper y .search-icon-btn para cada acción adyacente.

Clases y tokens

ClaseDescripción
.search-rowFlex wrapper para .search-bar + .search-icon-btn. La search bar ocupa el espacio disponible (flex:1).
.search-barPíldora --radius-full, fondo surface-container-high + borde var(--border). Íconos 24px. 40px cuando está dentro de .search-row (desktop). 56px standalone (mobile/hero).
.search-icon-btnBotón circular 40×40px para acciones adyacentes. Agregar .active / aria-pressed="true" para estado seleccionado. Siempre requiere aria-label.
.search-bar-iconÍcono de lupa (24×24px). Heredado de currentColor.
.search-bar-inputInput interno sin bordes. font-size-body-lg, caret-color: --color-secondary.
.search-bar-avatarAvatar circular opcional (30px). Fondo --color-primary-container.
.search-bar-trailingÍcono final opcional (micrófono, limpiar). 24×24px.
.search-viewVista expandida con resultados. Agregar .search-view-fullscreen en ventanas compactas.
