Auditoría: Search Icon Buttons

Embassy DS — Componente .search-icon-btn · Verificación contra Color Roles, States, y Button spec · Jun 2026

Resumen ejecutivo

El componente existe y tiene nombre propio: .search-icon-btn en search.css. No es un .icon-btn ni una variante de botón inventada. La forma circular (50%), el fill neutral por defecto, y el estado selected via --color-secondary-container están correctamente definidos. Lo que falta son 2 estados (pressed y disabled) y un ajuste en la opacidad del `:active`. La imagen del botón "relleno de azul sólido" que aparece en las capturas corresponde a un override fuera del DS — no es la definición canónica del default.

Hallazgos de la auditoría

✓

Forma circular correcta

.search-icon-btn usa border-radius: 50%. La forma circular no es una invención — está especificada explícitamente en el header de search.css: "Icon btn: 40×40px · border-radius 50% · ícono 20px". El .icon-btn de button.css usa --radius-md (8px) y NO es circular — si se usó ese clase, está mal mapeado.

✓

Background default: superficie neutral, no accent

El default correcto es --color-surface-container-high (neutral-50 / #EAEBED en light · #282C39 en dark). No lleva accent fill. Un botón de ícono con background accent sólido por defecto no existe en Embassy — ese tratamiento solo aparece en btn-primary (con label).

✓

Estado selected: secondary-container fill

El estado activo/seleccionado (filtro aplicado) usa --color-secondary-container como background, --color-secondary como borde, y --color-on-secondary-container como color de ícono. En light: periwinkle claro (#CAD9FF). En dark: mid-periwinkle (#3A5BB0). Se activa via [aria-pressed="true"] o clase .active. Alineado con los Color Roles.

✓

Focus ring

Implementado correctamente: outline: 2px solid var(--color-focus) + box-shadow: 0 0 0 4px var(--color-focus-ring). Idéntico al patrón compartido del resto del sistema (WCAG 2.4.7).

!

Estado pressed: solo opacity — inconsistente con el sistema

Actualmente: .search-icon-btn:active { opacity: .85 }. El resto del sistema usa un state layer sobre el fondo (container step o color-mix). La opacidad sola cambia toda la apariencia del botón (incluido el borde y el ícono) y no respeta el patrón de 12% de MD3. Propuesta: background: var(--color-surface-container-highest) — consistente con .icon-btn:active en button.css.

✗

Estado disabled: no definido

.search-icon-btn no tiene regla :disabled ni [aria-disabled]. El botón disabled hereda estilos del default, lo que no comunica el estado. Debe añadirse siguiendo el patrón de .icon-btn:disabled: background: transparent; border-color: var(--color-outline-variant); color: var(--color-on-disabled); cursor: not-allowed;

✗

Fondo "relleno accent" en las capturas → no pertenece al DS

El botón de campanilla (captura 1) muestra un fill azul/índigo sólido como estado default. En Embassy, ese tratamiento no existe para icon buttons. El único fill accent definido es el estado selected (--color-secondary-container, que es un periwinkle claro en light). Un fill --color-secondary (#4F80FF) o similar como default es un override fuera del DS que debe eliminarse o redefinirse como estado selected.

?

Borde var(--border) puede volverse invisible

--border es color-mix(in srgb, on-surface 10%, transparent) — un borde de contenedor, no de elemento interactivo. En algunos temas, el contraste visual es muy bajo. Para icon buttons interactivos, considerar --color-outline-variant (neutral-100 / #BFC1C8 en light) que es el borde sutil de elementos interactivos, con más cuerpo que el --border de contenedor.

Correcciones a aplicar en search.css

✗ actual Pressed & Disabled (actual) 
.search-icon-btn:active {
  opacity: .85; /* state layer débil */
}

/* ← sin regla :disabled */
✓ propuesto Pressed & Disabled (correcto) 
.search-icon-btn:active:not(:disabled) {
  background: var(--color-surface-container-highest);
  color: var(--color-on-surface);
}

.search-icon-btn:disabled,
.search-icon-btn[aria-disabled="true"] {
  background: transparent;
  border-color: var(--color-outline-variant);
  color: var(--color-on-disabled);
  cursor: not-allowed;
  pointer-events: none;
}
