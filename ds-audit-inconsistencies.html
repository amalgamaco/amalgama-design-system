§S1 Off-scale spacing values
✗ error

The spacing scale is a strict 4px grid: 4, 8, 12, 16, 20, 24, 32, 40, 48, 64, 80. Any value outside this set is not a named token and breaks the grid. The following values appear across the codebase and none of them map to a --space-* token.

✗ 6px — the most pervasive off-scale gap value

6px sits between --space-1 (4px) and --space-2 (8px) and is used as a gap/spacing unit in at least 6 components. It isn't defined as a token and creates a shadow mini-scale.

segmented-button.cssicon gap inside seg-btn6px--space-2 8px
Fix: Pick either --space-1 (4px) for tight icon separators or --space-2 (8px) for inline element gaps — chips and buttons already use --space-2 for their internal gaps. Add 6px as a token if the tighter step is intentional, or remove it in favor of 8px.
✗ 10px — the off-scale gap for "medium-density" contexts

10px sits between --space-2 (8px) and --space-3 (12px) and is used in at least 5 components as an "in-between" spacing, particularly for row/column gaps and badge padding.

§C1 Primitive token leakage in components
✗ error

The DS contract is that components consume only semantic tokens (--color-* roles). Primitives (--secondary-900, --error-600, etc.) should only be defined in variables.css. Components that reference primitives directly bypass the dark-mode recalibration layer.

✗ Avatar gradients use raw palette primitives — bypass semantic layer

Both vacancy-card.css and person-card.css define avatar backgrounds using raw primitive tokens in linear-gradient(). The semantic layer has no gradient tokens, so these gradients won't automatically adapt to dark mode recalibration.

§C4 Alias tokens used where semantic roles exist
⚠ warn

variables.css defines both semantic role tokens (--color-*) and shorthand alias tokens (--bg, --border, --accent, --interactive, --interactive-light). Aliases are convenience shortcuts for the docs shell and layouts — components should prefer the canonical role tokens to avoid two levels of indirection.
